# ArcadeBloom 主页游戏区域改造计划

## 项目概述
参考 CrazyGames 网站的设计理念，对 ArcadeBloom 主页的游戏区域进行全面改造，提升用户体验和视觉效果。

## CrazyGames 设计特点分析

### 1. 主页布局结构
- **顶部推荐区域**: "Top picks for you" - 横向滚动的游戏卡片
- **特色游戏区域**: "Featured games" - 大型游戏展示卡片，包含热门/更新标签
- **合作伙伴专区**: "Play on Comet" - AI浏览器推广区域
- **新游戏区域**: "New games" - 最新发布游戏展示
- **多类型游戏区域**: 按类型分区的游戏展示（多人游戏、各种类型等）
- **排行榜区域**: "Climb the Ranks" - 排行榜系统推广

### 2. 游戏卡片设计特点
- **极简卡片设计**: 只显示游戏图片 + 游戏名称（上方）
- **游戏标签**: 可选的Hot、New、Updated、Top 等状态标签
- **悬停效果**: 卡片悬停时的缩放效果
- **游戏截图**: 高质量的游戏截图展示
- **去除冗余信息**: 不显示评分、播放次数、描述等额外信息

### 3. 用户体验设计
- **分类导航**: 左侧边栏的分类导航，包含图标
- **搜索功能**: 顶部搜索框，支持游戏搜索
- **横向滚动**: 各区域支持横向滚动浏览更多游戏
- **响应式设计**: 适配不同屏幕尺寸

## 当前 ArcadeBloom 问题分析

### 1. 布局问题
- 游戏展示过于简单，缺乏层次感
- 缺少明确的游戏分类和推荐区域
- 游戏卡片信息过于复杂，需要简化
- 缺乏游戏状态标识（新游戏、热门等）

### 2. 功能缺失
- 没有横向滚动的游戏区域
- 游戏标签和分类展示不够突出
- 游戏卡片设计过于复杂，需要简化为CrazyGames风格

## 改造方案

### 第一阶段：布局重构

#### 1.1 头部区域优化
- 保持现有导航栏不变
- 优化搜索框设计，增加搜索建议功能
- 保持简洁的无用户登录设计，专注于游戏体验

#### 1.2 侧边栏分类导航设计

**侧边栏导航结构**
```
侧边栏布局：
┌─ 🏠 Home (首页)              ── 默认显示 Featured Games
├─ 🕐 Recently Played          ── 用户最近玩过的游戏（本地存储）
├─ ⭐ Featured Games           ── 编辑精选的特色游戏
├─ ⚔️ Action                   ── 动作游戏分类
├─ 🗺️ Adventure                ── 冒险游戏分类
├─ 🕹️ Arcade                   ── 街机游戏分类
├─ 💎 Bejeweled                ── 消除类游戏分类
├─ 👦 Boys                     ── 男孩游戏分类
├─ 👆 Clicker                  ── 点击游戏分类
├─ 👧 Girls                    ── 女孩游戏分类
├─ 📱 Hypercasual              ── 超休闲游戏分类
├─ 🧩 Puzzle                   ── 益智游戏分类
├─ ♟️ Strategy                 ── 策略游戏分类
├─ 🌴 Casual                   ── 休闲游戏分类
├─ ⚽ Sports                   ── 体育游戏分类
└─ 🏎️ Racing                   ── 赛车游戏分类
```

**侧边栏设计特点：**
- 参考CrazyGames的侧边栏设计
- 每个分类配有对应图标
- 当前选中分类高亮显示
- 支持折叠/展开（移动端）
- 响应式设计，移动端可隐藏为汉堡菜单

#### 1.3 主内容区域设计

**A. Featured Games (首页默认显示)**
```
设计特点：
- 大型游戏卡片 (2x2 网格布局)
- 包含游戏标签：UPDATED
- 简化的卡片设计：图片 + 游戏名称
- 可选的状态标签（Hot、Updated、Top）
- 横向滚动展示更多游戏
```

**B. 分类游戏页面**
```
设计特点：
- 根据侧边栏选择的分类显示对应游戏
- 网格布局展示该分类下的所有游戏
- 简化的游戏卡片：图片 + 名称
- 支持排序功能（按热度、名称等）
- 横向滚动或分页加载更多游戏
```

**C. Recently Played**
```
设计特点：
- 显示用户最近玩过的游戏（基于本地存储）
- 按游戏时间倒序排列
- 简化的卡片设计：图片 + 游戏名称
- 快速继续游戏功能
```



### 第二阶段：游戏卡片重设计（CrazyGames风格）

#### 2.1 卡片布局标准化
```
卡片尺寸规范：
- 小卡片：150x200px (网格布局)
- 中等卡片：200x250px (特色游戏)
- 大卡片：400x300px (主推荐游戏)

CrazyGames风格卡片内容：
- 游戏截图 (高质量，统一比例)
- 游戏名称 (显示在图片上方)
- 可选的状态标签 (Hot/Updated/New/Top)
- 去除：评分、播放次数、描述、分类标签等冗余信息
```

#### 2.2 交互效果
```
保持现有交互效果：
- 卡片悬停效果（当前的上移动画）
- 阴影效果
- 图片缩放效果
- 按钮过渡效果

点击效果：
- 保持现有的新标签页打开游戏方式
```

### 第三阶段：功能增强

#### 3.1 搜索功能增强
```
搜索特性：
- 实时搜索建议
- 游戏名称搜索
- 分类搜索
- 本地搜索历史记录（浏览器存储）
```

### 第四阶段：视觉优化

#### 4.1 配色方案
```
主色调：
- 保持现有紫色渐变主题
- 优化对比度，提升可读性
```

#### 4.2 图标和标签
```
状态标签设计：
- UPDATED: 蓝色渐变，刷新图标
- TOP: 金色渐变，皇冠图标
```

#### 4.3 动画效果
```
页面加载：
- 渐进式加载动画
- 卡片淡入效果
- 平滑滚动

交互反馈：
- 按钮点击动画
- 加载状态指示器
- 成功/错误提示
```
